# Agent Handoff Guide

This file is the first stop for Codex, Claude Code, and other coding agents working on ReadMark.

## Project Identity

- Product name: `ReadMark`
- Obsidian plugin ID: `readmark`
- Package name: `readmark`
- GitHub remote: `https://github.com/Amazinnn/ReadMark.git`
- Stable project root: `D:\Projects\项目仓库Bingo\.obsidian\plugins\readmark`
- Local Obsidian test vault: `D:\Projects\项目仓库Bingo`

Obsidian plugin IDs must be lowercase. Keep `manifest.json.id` as `readmark`; use `ReadMark` for the human-facing product name.

## Read Before Editing

Read these files before changing behavior:

- `CONTEXT.md` for domain terms and product language.
- `docs/ARCHITECTURE.md` for plugin structure and timing flow.
- `docs/DATA_MODEL.md` before touching storage, sessions, excerpts, or migration logic.
- `docs/DEVELOPMENT.md` before changing setup, build, QA, or release workflow.
- `docs/VERSIONING.md` before changing versions, tags, release artifacts, or compatibility.

## Source Boundaries

Primary source files:

- `src/main.ts` - Obsidian plugin logic, DOM rendering, timing, scanning, storage.
- `styles.css` - sidebar UI styling.
- `manifest.json` - Obsidian plugin metadata.
- `package.json` - build scripts and package metadata.

Build artifact:

- `main.js` is generated by `npm run build`. Do not edit it by hand.

Runtime/private data:

- `data.json`
- `books/`
- `node_modules/`

Never commit runtime reading data. It can contain local paths, reading sessions, progress, and excerpt text.

## Development Commands

Install dependencies:

```powershell
npm install
```

Build:

```powershell
npm run build
```

This repository lives directly in the local Obsidian test vault's plugin directory. After `npm run build`, reload the plugin or restart Obsidian to test the new build.

## Required Verification

After code changes, run:

```powershell
npm run build
```

Manual checks should cover:

- Open ReadMark from the ribbon/command palette.
- Add or open a tracked Markdown book.
- Confirm timing starts only when the central Obsidian area shows the book.
- Confirm switching to the sidebar does not stop timing.
- Confirm closing or replacing the central book stops timing.
- Confirm highlights and annotations update after source edits.
- Confirm the sidebar renders current book, bookshelf, and stats without collapsed-panel flicker.

## Implementation Constraints

- Keep storage local-first and JSON-based.
- Do not store timer tick events.
- Treat reading records as user-owned data.
- Treat excerpt records as source-derived snapshots unless a future design explicitly adds saved excerpts.
- Prefer small, focused changes over broad rewrites.
- Do not move work back into the old plugin directory `markdown-reading-tracker`.
- Keep version fields synchronized across `manifest.json`, `package.json`, `package-lock.json`, and `versions.json`.
